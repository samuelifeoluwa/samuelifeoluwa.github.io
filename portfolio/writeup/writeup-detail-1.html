Introduction

Overview of basic payload obfuscation techniques used to reduce detection while maintaining functionality. Includes backtick splitting, variable renaming, and string concatenation methods.

Technique 1: Backtick Character Splitting

Original: New-Object
Obfuscated: N`e`w-`O`b`j`e`c`t
Original: Out-String
Obfuscated: O`u`t-`S`t`r`i`n`g

PowerShell treats backticks as escape characters but ignores them when parsing cmdlets.

Technique 2: Variable Name Obfuscation

Original: $client = New-Object System.Net.Sockets.TCPClient
Obfuscated: $fglarowugbycc = N`e`w-`O`b`j`e`c`t System.Net.Sockets.TCPClient

Using random, non-descriptive variable names avoids signature-based detection.

Technique 3: String Concatenation

Original: "PowerShell"
Obfuscated: "Pow" + "er" + "Shell"

Breaking strings into multiple parts prevents simple pattern matching.

Conclusion

Combining multiple obfuscation techniques significantly increases the chance of bypassing signature-based AV solutions. However, modern EDR solutions with behavioral analysis may still detect malicious activity.
