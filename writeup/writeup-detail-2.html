Web Application Security Findings

Web Security Misconfigurations Bug Bounty API Security

Overview

Summary of real-world security misconfigurations discovered during vulnerability assessments, including exposed API keys, missing security headers, and IDOR vulnerabilities.

Finding 1: Exposed API Keys

API keys were found in client-side JavaScript code and public GitHub repositories. This allowed unauthorized access to backend services.

Example: const API_KEY = "sk_live_12345abcdef67890";

Finding 2: Missing Security Headers

Common security headers missing:

  • Content-Security-Policy (CSP)
  • X-Frame-Options
  • Strict-Transport-Security (HSTS)
  • X-Content-Type-Options

Finding 3: IDOR Vulnerabilities

Insecure direct object references allowed access to other users' data by modifying numeric IDs in API requests.

GET /api/user/1234 → Victim's data accessible by changing ID to 1235

Finding 4: Public Storage Buckets

Misconfigured AWS S3 buckets allowed public read/write access to sensitive user-uploaded content.

Recommendations

  • Never store secrets in client-side code
  • Implement proper security headers
  • Use UUIDs instead of sequential IDs
  • Configure cloud storage with proper ACLs
